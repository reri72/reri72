* shared Memory 는 두 개 이상의 프로세스가 메모리 영역을 공유하여 데이터를 빠르게 교환할 수 있도록 하는 기법이다.
* 속도가 다른 IPC 기법에 비해 빠르고. 대용량 데이터를 처리하는 데 유리하다.
* 시스템의 메모리 공간을 프로세스 간에 공유하는 방식으로 동작한다.
* 여러 프로세스가 동시에 메모리 영역을 수정할 수 있기 때문에 동기화 메커니즘이 필요하다.

* shmget
 > 공유메모리 세그먼트를 생성하거나 접근

* shmat
 > 공유메모리 세그먼트를 현재의 프로세스의 주소 공간에 첨부

* shmdt
 > 공유메모리 세그먼트를 분리

*shmctl
 > 공유메모리 세그먼트에 대한 제어 작업 (삭제 등)


* 예제에서는 write가 먼저 쓰고 read가 읽는 절차를 가진다.
* 만일 코드 상 공유메모리 세그먼트 삭제가 안되었다면
 > ]# ipcs -m
 > ]# ipcrm -m (공유메모리 아이디)
 명령어를 수행해주면 삭제할 수 있다.
 공유메모리 세그먼트를 삭제를 하기 전에는 사용중인 프로세스를 모두 종료하여야 한다.
